## 2026-05-11 - [CRITICAL] Fixed Hardcoded Firebase Secrets & Added Secure Backend Layer
**Vulnerability:** Firebase API keys and configuration were hardcoded directly in all 9 HTML files on the frontend, exposing them to any user who viewed the source code. The application lacked a backend and rate limiting, leaving it vulnerable to abuse.
**Learning:** While Firebase config on the frontend isn't strictly fully sensitive if Firebase Security Rules are correctly implemented, having a backend layer to serve this config adds an extra layer of security, prevents automated scrapers from easily grabbing the keys, and allows for rate-limiting (preventing abuse/crashing under high load).
**Prevention:** Never hardcode API keys or configuration in frontend source code. Always use a secure backend layer (e.g., Express) to serve configuration via an endpoint protected by rate-limiting (e.g., `express-rate-limit`) and proper CORS/Helmet security headers. Store secrets securely in a `.env` file that is added to `.gitignore`.
